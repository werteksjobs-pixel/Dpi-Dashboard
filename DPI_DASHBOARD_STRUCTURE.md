# 🏗️ Структура проекта DPI Dashboard

Этот файл описывает основные компоненты интерфейса и логику взаимодействия между слоями приложения.

---

## 📂 Основные файлы отображения

### 1. `index.html` (Каркас интерфейса)
Это главный файл разметки. Он разделен на несколько ключевых секций:
- **Header / Titlebar**: Содержит название приложения и кнопки управления (Свернуть, Закрыть, Настройки, **Мониторинг**).
- **Cards (Карточки сервисов)**: 
  - `card-zapret`: Управление службой Zapret (Bypass).
  - `card-tgproxy`: Управление Telegram Proxy.
- **Modals (Модальные окна)**:
  - `stats-modal`: Центр мониторинга (график трафика, история статусов, Speed Test).
  - `app-modal`: Настройки автозапуска, темы и языка.
  - `dns-modal`: Тест DNS-утечек.
  - `list-modal`: Редактор списка доменов `list-general.txt`.
  - `scan-modal`: Автоматический подбор лучшей стратегии обхода.

### 2. `src/renderer.ts` (Логика UI)
Этот файл управляет поведением всех элементов, которые вы видите на экране. Основные блоки:
- **`els` Object**: Массив ссылок на все DOM-элементы (кнопки, поля ввода, окна). Если вы добавили новый элемент в HTML с ID, его нужно прописать здесь.
- **Service Controls**: Функции `toggle-zapret` и `toggle-tgproxy`. Они отправляют команды в `main.ts` и меняют стиль карточек (OFFLINE -> ONLINE).
- **Monitoring Logic**:
  - `drawTrafficChart()`: Отрисовывает живой график скорости на Canvas.
  - `onTrafficStats`: Слушатель данных о скорости, которые приходят из фонового процесса.
  - `onStatusEvent`: Обновляет "Историю статусов" при падении или восстановлении связи.
- **Speed Test**: Обработка кнопки "ЗАПУСТИТЬ ТЕСТ", взаимодействие с API Cloudflare через фоновый процесс.
- **I18n**: Система локализации. Переключает тексты между русским и английским языками без перезагрузки страницы.

### 3. `src/preload.ts` (Мост безопасности)
Связующее звено между UI и системными функциями. Он открывает доступ к объекту `window.electronAPI`, через который `renderer.ts` может:
- Запускать процессы.
- Читать/писать файлы настроек.
- Получать статистику трафика.

---

## 🔄 Как добавить новый элемент?

1. Добавьте HTML-разметку в `index.html` с уникальным `id`.
2. В `src/renderer.ts` добавьте этот элемент в объект `els`.
3. Опишите логику (например, `onclick`) внутри `window.addEventListener('DOMContentLoaded', ...)` в `renderer.ts`.
4. Если нужно выполнить системное действие (например, запустить новый EXE), добавьте соответствующий канал в `preload.ts` и обработчик в `main.ts`.

## 🎨 Стилизация (index.html <style>)
Мы используем **Glassmorphism Design System**:
- **Переменные CSS**: Все цвета (`--accent-blue`, `--bg-card` и др.) вынесены в начало тега `<style>` для удобной смены тем.
- **Анимации**: Используются плавные переходы (`transition`) и ключевые кадры (`@keyframes`) для эффектов пульсации статусов и появления окон.
