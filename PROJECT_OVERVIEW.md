# DPI Dashboard: Технический Обзор Проекта

## 1. О проекте (Overview)
**DPI Dashboard** — это настольное Windows-приложение на базе Electron, предназначенное для удобного управления утилитами обхода глубокого анализа пакетов (DPI) и проксирования трафика. 

Приложение решает проблему сложности настройки CLI-инструментов (таких как Zapret), предоставляя современный графический интерфейс для:
- Управления процессами обхода DPI (`winws.exe`).
- Развертывания локального MTProto WebSocket прокси для Telegram (`tg_ws_proxy.exe`).
- Автоматического тестирования и выбора наиболее эффективных стратегий обхода.
- Модификации системного файла `hosts` для обхода DNS-блокировок.

## 2. Стек технологий
Проект построен на современном стеке веб-технологий и инструментов системного администрирования:
- **Core**: Electron (v31+) — фреймворк для кроссплатформенных приложений.
- **Language**: TypeScript — обеспечивает типизацию IPC-каналов и системной логики.
- **Frontend**: Vanilla HTML5 / CSS3 (Custom Properties, Glassmorphism) — для максимальной производительности без тяжелых фреймворков.
- **Build System**: Electron Builder — упаковка в NSIS-установщик с поддержкой запроса прав администратора.
- **Networking**: Node.js `child_process` (управление бинарниками), `net` и `dns` (тестирование соединений).
- **Updates**: `electron-updater` — механизм ручного обновления через GitHub Releases.

## 3. Архитектура приложения

Приложение разделено на три основных слоя согласно стандартам безопасности Electron:

### 3.1 Main Process (`src/main.ts`)
Центральный оркестратор, выполняющий следующие функции:
- **Жизненный цикл**: Управление окнами, Tray-меню и Single Instance Lock (запрет запуска копий).
- **Security Enforcement**: Проверка прав администратора при старте (`net session`). В упакованном виде приложение блокирует запуск без повышенных привилегий.
- **Process Orchestration**: Spawn и Kill дочерних CLI-утилит.
- **Auto-Update**: Логика `autoUpdater` с отключенной автоматической загрузкой (`autoDownload = false`) для реализации ручного управления.
- **Resource Management**: Распаковка бинарных ресурсов из ASAR во временную папку AppData для корректного исполнения.

### 3.2 Preload Layer (`src/preload.ts`)
Безопасный мост (ContextBridge), который:
- Изолирует Main процесс от Renderer.
- Экспонирует строго определенный набор IPC-методов через объект `window.electronAPI`.
- Обеспечивает типизацию событий для фронтенда.

### 3.3 Renderer Process (`src/renderer.ts` & `index.html`)
Отвечает за UI и взаимодействие с пользователем:
- **State Management**: Сохранение настроек пользователя в `localStorage`.
- **i18n**: Встроенная система интернационализации (RU/EN).
- **Theme Engine**: Динамическое переключение тем (Space, Dark, Light) через CSS-переменные.
- **Update UI**: Логика отображения модального окна обновления и прогресс-бара загрузки.

### 3.4 IPC Каналы (Events)
| Канал | Направление | Описание |
|---|---|---|
| `toggle-zapret` | R → M | Запуск/Остановка winws.exe с параметрами |
| `toggle-tgproxy` | R → M | Запуск/Остановка прокси для Telegram |
| `status` | M → R | Передача статуса процесса (Running/Stopped) |
| `log` | M → R | Передача строк `stdout`/`stderr` в терминал UI |
| `get-app-version` | R → M | (Invoke) Получение версии из package.json |
| `update-status` | M → R | Уведомление о наличии новой версии на GitHub |
| `download-update` | R → M | Команда на начало скачивания обновления |
| `hosts-apply` | R → M | Запись выбранных IP/Доменов в системный hosts |

## 4. Управление процессами (Core Logic)
Приложение взаимодействует с внешними утилитами через модуль `child_process`:
1. **Zapret (`winws.exe`)**:
   - Аргументы формируются динамически на основе выбранной стратегии (например, `fake-tls-pro`).
   - Используется флаг `--wf-l3addr` для фильтрации трафика.
   - Логи читаются через `stdout.on('data')` и пересылаются в Renderer в реальном времени.
2. **Telegram Proxy (`tg_ws_proxy.exe`)**:
   - Запускается как WebSocket сервер на выбранном порту.
   - Поддерживает маскировку под Fake TLS.
   - Генерирует ссылку `https://t.me/proxy?...` для быстрой настройки мессенджера.

## 5. Работа с данными
- **`assets/hosts-data.json`**: База данных доменов и IP для редиректора (Discord, OpenAI, Claude и др.).
- **`lists/list-general.txt`**: Список доменов для селективного обхода (используется Zapret).
- **`ips.json` / `ips.txt`**: Служебные файлы для хранения кешированных IP-адресов.
- **Config**: Пользовательские настройки сохраняются в `AppData/dpi-dashboard/config.json`.

## 6. Системные интеграции и Безопасность
- **UAC (User Account Control)**: В манифест сборщика добавлен параметр `requestedExecutionLevel: requireAdministrator`. При запуске `main.ts` дополнительно проверяет привилегии через вызов `net session`.
- **System Tray**: При закрытии главного окна приложение не завершается, а сворачивается в трей. Доступно быстрое управление через контекстное меню.
- **Update Flow**: 
  1. `checkForUpdates()` при старте или по кнопке.
  2. Если доступно — Renderer показывает кастомную модалку.
  3. При нажатии "Да" — `downloadUpdate()`.
  4. По завершении — автоматический `quitAndInstall()`.

## 7. Инструкция для разработчиков (Dev Guide)

### Установка зависимостей
```bash
npm install
```

### Запуск в режиме разработки
Для корректной работы сетевых функций требуется запуск от админа:
```bash
npm run dev:admin
```
*(Использует PowerShell для повышения прав процесса Electron)*

### Сборка приложения
```bash
# Сборка готового инсталлера (.exe)
npm run dist
```
Готовый файл появится в папке `dist/`.

### Структура папок
- `/src`: Исходный код TypeScript (Main, Preload, Renderer).
- `/bin`: Бинарные файлы `winws.exe`, `windivert.dll`, `tg_ws_proxy.exe`.
- `/assets`: Иконки, JSON-базы данных.
- `/lists`: Текстовые файлы списков доменов.
- `/build`: Результаты компиляции TS и ресурсы для сборщика.
